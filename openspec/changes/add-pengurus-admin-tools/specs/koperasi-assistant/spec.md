## ADDED Requirements

### Requirement: Data-grounded assistant chat for Pengurus

The system SHALL provide a Pengurus-only chat endpoint that answers questions about the
cooperative using a Google Gemini model, grounded on a maintained snapshot of database
metadata and aggregates (member/loan/savings/renteng statistics). The model SHALL be
configurable via environment (any Gemini model) without code changes.

#### Scenario: Assistant answers from cooperative data

- **WHEN** a Pengurus asks the assistant a question about the cooperative's data
- **THEN** the response is generated by the configured Gemini model using the current metadata/aggregate snapshot as grounding

#### Scenario: Non-Pengurus is denied

- **WHEN** a non-Pengurus calls the assistant endpoint
- **THEN** the request is rejected with a forbidden error

### Requirement: Periodically refreshed metadata snapshot

The system SHALL compute and store a snapshot of DB metadata + aggregates, refresh it
periodically, and reuse the cached snapshot between refreshes. It SHALL use Redis when
configured and fall back to an in-memory cache otherwise.

#### Scenario: Snapshot is cached and reused

- **WHEN** two assistant requests occur within one refresh interval
- **THEN** both reuse the same cached snapshot without recomputing it each time

#### Scenario: Redis-optional degradation

- **WHEN** Redis is not configured
- **THEN** the snapshot is cached in memory and the assistant still functions

### Requirement: Optional external grounding

The system SHALL support enriching answers with external/web context via Gemini grounding when
enabled, without breaking answers when it is disabled or unavailable.

#### Scenario: External grounding toggled off

- **WHEN** web grounding is disabled or unavailable
- **THEN** the assistant still answers from the cooperative data snapshot
